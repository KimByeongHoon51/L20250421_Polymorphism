int A           instance

언리얼에서 Actor 클래스를 가진다 = transform을 할 수 있다.

실습:
세상이 있습니다. 세상에는 플레이어와 몬스터가 있습니다.
몬스터는 고블린, 슬라임, 멧돼지가 있습니다.
고블린은 달릴 수 있고 공격 가능합니다.
슬라임을 미끄러지고 공격 가능합니다.
멧돼지는 들이박고 공격 가능합니다.
플레이어는 HP와 MP가 있고 공격이 가능하고 몬스터를 잡아서 골드를 모을 수 있습니다.

명사: 세상, 플레이어, 몬스터, 고블린, 슬라임, 멧돼지, HP, MP, 골드
동사: 공격한다, 달린다, 미끄러진다, 들이박는다, 모으다, 잡다(Die, Drop)

접근 제한자
		같은 클래스 	같은 패키지	다른 패키지의 하위 클래스 	다른 패키지
-public 	    O		    O			   O			     O
-protected 	    O		    O			   O			     X
-private 	    O		    X			   X			     X

제일 먼저 할 것. 명사, 동사 뽑아서 클래스 만들기
base class = parent class = 부모 클래스
derived class = 자식 클래스

ISA 관계 (is a) (상위 클래스 하위 클래스 관계라고도 함)
-ex)Slime is a Monster.

HAS-A관계 (has a)
-Player has a Inventory

USES-A 관계 (use a)
-Skill uses a Enemy

단단한 ISA 결합보다 유연한 연결 HAS-A, USES-A 관계도 있고 이는 확장성을 높인다.

부모 클래스와 자식 클래스 만들 때는 부모 클래스 먼저 설계
공통 인터페이스 -> 구체화 -> 특수화
